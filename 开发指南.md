## 核心架构

### 插件主类 GameFunXiao.java

插件主类负责初始化所有管理器，初始化顺序如下：

```
ConfigManager → MessageManager → WorldManager → PlayerDataManager
→ LeaderboardManager → RoomManager → GameManager → MenuManager
→ 注册命令 → 注册监听器
```

获取各管理器实例：

```java
GameFunXiao plugin = GameFunXiao.getInstance();

plugin.getConfigManager();      // 配置管理
plugin.getMessageManager();     // 消息管理
plugin.getWorldManager();       // 世界管理
plugin.getPlayerDataManager();  // 玩家数据管理
plugin.getLeaderboardManager(); // 排行榜管理
plugin.getRoomManager();        // 房间管理
plugin.getGameManager();        // 游戏逻辑管理
plugin.getMenuManager();        // 菜单管理
```

## 配置系统

### ConfigManager

负责读取 `config.yml` 和 `config/` 文件夹下的所有配置文件。

```java
// 获取主配置
FileConfiguration config = plugin.getConfigManager().getConfig();

// 获取子配置文件（如 gamemodes.yml）
FileConfiguration gamemodes = plugin.getConfigManager().getConfig("gamemodes");

// 获取通用前缀
String prefix = plugin.getConfigManager().getPrefix();

// 获取猎人游戏前缀
String hgPrefix = plugin.getConfigManager().getHunterGamePrefix();

// 重载所有配置
plugin.getConfigManager().reloadConfigs();
```

### MessageManager

负责读取 `messages.yml`，所有消息都通过此类发送。

```java
// 获取消息（不带前缀）
String msg = plugin.getMessageManager().getMessage("room.created");

// 获取消息（带通用前缀）
String msg = plugin.getMessageManager().getMessageWithPrefix("general.reload_success");

// 获取消息（带猎人游戏前缀）
String msg = plugin.getMessageManager().getHunterGameMessageWithPrefix("game.started");

// 带变量替换
Map<String, String> placeholders = new HashMap<>();
placeholders.put("player", player.getName());
placeholders.put("room_id", roomId);
String msg = plugin.getMessageManager().getHunterGameMessageWithPrefix("room.created", placeholders);

// 直接发送给玩家
plugin.getMessageManager().sendMessage(player, "general.no_permission");
plugin.getMessageManager().sendMessage(player, "room.player_joined", placeholders);
```

**消息路径规范：**
- `general.*` - 通用消息，使用通用前缀
- `room.*` - 房间相关消息，使用猎人游戏前缀
- `game.*` - 游戏相关消息，使用猎人游戏前缀
- `lobby.*` - 大厅相关消息，使用猎人游戏前缀

## 菜单系统

### 创建新菜单

所有菜单必须继承 `BaseMenu`：

```java
public class MyMenu extends BaseMenu {

    public MyMenu(GameFunXiao plugin, Player player) {
        super(plugin, player, "§0§l菜单标题", 54); // 54格菜单
    }

    @Override
    protected void setupItems() {
        inventory.clear();

        // 填充米饭风格边框
        fillMiFanBorder();

        // 标题物品（放在第5格，索引4）
        inventory.setItem(4, createTitleItem(Material.COMPASS,
            "§e菜单标题",
            "§7副标题说明"
        ));

        // 功能按钮
        inventory.setItem(22, createItem(Material.EMERALD,
            "   §8[§a功能按钮§8]",
            "§8· · · · · · · · · · · · · ·",
            "§f- §7点击执行某操作",
            "§8· · · · · · · · · · · · · ·"
        ));

        // 返回按钮（左下角，索引45）
        inventory.setItem(45, createBackButton());

        // 关闭按钮（右下角，索引53）
        inventory.setItem(53, createCloseButton());
    }

    @Override
    public void handleClick(InventoryClickEvent event) {
        // 注意：MenuListener 已经在调用此方法前设置了 event.setCancelled(true)
        int slot = event.getRawSlot();
        if (slot < 0 || slot >= inventory.getSize()) return;

        switch (slot) {
            case 22 -> {
                playClickSound();
                // 执行操作
            }
            case 45 -> {
                playClickSound();
                plugin.getMenuManager().openMainMenu(player);
            }
            case 53 -> {
                playCloseSound();
                player.closeInventory();
            }
        }
    }
}
```

### 打开菜单

在 `MenuManager` 中注册并添加打开方法：

```java
// MenuManager.java 中添加
public void openMyMenu(Player player) {
    MyMenu menu = new MyMenu(plugin, player);
    openMenu(player, menu);
}
```

### 菜单监听器注意事项

`MenuListener.java` 在调用 `menu.handleClick(event)` 之前已经设置了 `event.setCancelled(true)`，所以在 `handleClick` 里不需要再次取消事件。

```java
// MenuListener.java 核心逻辑
@EventHandler
public void onInventoryClick(InventoryClickEvent event) {
    // 先取消事件，防止物品被拿走
    event.setCancelled(true);

    BaseMenu menu = plugin.getMenuManager().getOpenMenu(player);
    if (menu != null) {
        menu.handleClick(event);
    }
}
```

## 游戏系统

### 房间状态流转

```
WAITING（等待中）
    ↓ 人数达到 min_players
STARTING（倒计时中）
    ↓ 倒计时结束
SELECTING（世界选择中）
    ↓ 猎物确认世界
PLAYING（游戏中）
    ↓ 猎物完成目标 或 猎人击杀猎物
ENDED（已结束）
```

## 世界系统

### WorldManager 说明

插件使用独立世界系统，每个房间有自己的大厅世界和游戏世界：

- **模板大厅世界** (`hugamelobby`) - 管理员编辑的大厅模板
- **房间大厅世界** (`gamefun_lobby_<roomId>`) - 从模板复制的房间大厅
- **游戏主世界** (`gamefun_game_<roomId>`) - 游戏主世界（主世界）
- **游戏下界** (`gamefun_game_<roomId>_nether`) - 游戏下界
- **游戏末地** (`gamefun_game_<roomId>_end`) - 游戏末地

```java
// 获取大厅世界
World lobby = plugin.getWorldManager().getLobbyWorld(roomId);

// 创建大厅世界
World lobby = plugin.getWorldManager().createLobbyWorld(roomId);

// 创建游戏世界
World game = plugin.getWorldManager().createGameWorld(roomId);

// 创建下界和末地
plugin.getWorldManager().createGameWorldDimensions(roomId);

// 启用游戏世界规则（游戏开始时调用）
plugin.getWorldManager().enableGameWorldRules(world);

// 删除游戏世界
plugin.getWorldManager().deleteGameWorlds(roomId);

// 删除大厅世界
plugin.getWorldManager().deleteLobbyWorld(roomId);

// 获取模板大厅世界
World template = plugin.getWorldManager().getTemplateLobbyWorld();
```

## 颜色规范

项目使用 `§x§R§R§G§G§B§B` 格式的十六进制颜色，以下是常用颜色：

| 用途 | 颜色代码 | 效果 |
|------|----------|------|
| 成功 | `§x§5§5§F§F§A§A` | 青绿色 |
| 错误 | `§x§F§F§5§5§5§5` | 红色 |
| 警告 | `§x§F§F§A§A§5§5` | 橙色 |
| 信息 | `§x§8§8§D§D§F§F` | 蓝色 |
| 金币 | `§x§F§F§D§7§0§0` | 金色 |
| 猎人游戏渐变 | `§x§F§F§6§6§0§0` → `§x§C§C§F§F§9§9` | 橙色到青色 |

消息颜色规范（米饭风格）：
- `§8` - 深灰色（方括号 `[` `]`）
- `§7` - 灰色（普通说明文字）
- `§f` - 白色（短横线 `-`）
- `§a` - 绿色（成功/确认）
- `§c` - 红色（取消/警告）
- `§b` - 青色（数值/价格）
- `§e` - 黄色（提示）

## 注意事项

1. **菜单物品防拿取** - `MenuListener` 已在调用 `handleClick` 前设置 `event.setCancelled(true)`，不要在子菜单里重复设置
2. **消息前缀** - `room.*`、`game.*`、`lobby.*` 消息使用 `getHunterGameMessageWithPrefix()`，`general.*` 使用 `getMessageWithPrefix()`
3. **配置路径** - 读取配置时注意路径层级，如 `game.survival_time` 而不是 `survival_time`
4. **世界操作** - 世界创建是耗时操作，尽量使用异步处理，参考 `GameManager.startCountdown()` 中的异步世界创建
5. **玩家状态恢复** - 游戏结束时必须恢复玩家的位置、背包、血量、饱食度、经验，参考 `GameManager.endGame()` 中的恢复逻辑
6. **构建** - 使用 `mvn clean package` 构建，产物在 `target/gamefunxiao-1.0.0.jar`
